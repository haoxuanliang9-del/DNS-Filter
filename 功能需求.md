# 一、功能需求
## DNS协议解析
1.报文头部解析与构造
2.查询字段的域名编码与解码：正确处理指针压缩
3.查询字段的查询类型解析：决定处理还是透明转发
4.拦截报文的完整构造
解析出查询类型和opcode，只处理标准查询且A或AAAA的查询报文，其它透明转发。
解析出域名，在黑名单中查找，若存在于黑名单做下一步构造拦截报文，否则透明转发。
构造拦截报文后发送给客户程序。

# 三、架构设计
网络层：
  - epoll网络模型类：负责事件循环和UDP socket管理
  - UDP服务器类：接收客户端请求，返回响应

核心层：
  - DNS报文解析类：解析DNS消息，提取域名
  - DNS服务器类：协调各个模块，处理查询流程
  - 上游DNS转发类：与上游DNS服务器通信

业务层：
  - 过滤引擎类：检查域名是否在黑名单中，决定拦截或放行
  - 规则管理类：加载规则文件，实现热更新
  - 日志类：记录查询日志、拦截日志、错误日志
  - 配置管理类：读取和管理配置文件
  - 统计监控类：收集查询统计、拦截统计
  - HTTP API服务类：为GUI提供接口

表现层：
  - QT界面：通过HTTP API与核心程序通信
